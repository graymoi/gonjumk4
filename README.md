## 一、我们是谁

城乡建设咨询领域的政策知识服务系统，由用户与AI智能体协作运行。

---

## 二、核心目标

**最终目标**：将政策洞察转化为可执行的项目方案，解锁下游设计和施工合同

**价值链**：
```
政策采集 → 政策解读 → 项目谋划 → 资金匹配 → 方案输出 → 设计合同
```

**核心目标分解**：

| 目标 | 说明 | 支撑场景 |
|------|------|----------|
| 赚钱（设计合同收入） | 精准谋划、设计合同转化 | 场景1、2、3 |
| 赚钱（咨询费收入） | 智能筛选、战略规划、深度研究 | 场景4、5、6、7、9 |
| 增值（全流程服务） | 项目谋划→设计合同→建设服务 | 场景1、2、3 |
| 增值（高附加值服务） | 战略规划、深度研究、流程指导 | 场景5、9 |
| 省人力（AI全程辅助） | AI辅助、智能推荐 | 所有场景 |
| 省人力（自动化） | 自动采集、自动验证、自动更新 | 场景11 |

## 四、11个具体场景

| 场景 | 核心需求 | 收入来源 | 触发方式 |
|------|---------|----------|----------|
| 场景1：项目全流程谋划 | 精准谋划→设计合同→建设服务 | 设计合同收入 | `/谋划 项目名称` |
| 场景2：多个项目组合谋划 | 多项目组合，提高申报成功率 | 设计合同收入 | `/组合 项目列表` |
| 场景3：项目打捆申报 | 多项目打包申报，提高成功率 | 设计合同收入 | `/打捆 项目列表` |
| 场景4：政府资金申请项目筛选 | 智能筛选项目，优先级排序 | 咨询费收入 | `/筛选 项目列表` |
| 场景5：未发布/模糊政策深度研究 | 深入研究政策线索和趋势 | 咨询费收入 | `/研究 政策关键词` |
| 场景6：资金政策流程流向指导 | 指导如何拿到资金 | 咨询费收入 | `/指导 资金类型` |
| 场景7：特殊项目合同依据咨询 | 提供合同依据和合规性分析 | 咨询费收入 | `/咨询 项目描述` |
| 场景8：快速组装宣讲材料 | 快速组装PPT和宣讲材料 | 提高效率 | `/宣讲 主题` |
| 场景9：城市项目战略规划与拼盘 | 战略规划+拼盘+打捆 | 咨询费收入 | `/规划 城市 项目列表` |
| 场景10：知识库快速组装 | 快速组装知识库零件 | 提高效率 | `/组装 关键词` |
| 场景11：组织架构数据管理 | 管理城市组织架构和联系方式 | 基础资产 | `/管理 数据类型` |

## 八、Rules应用

### 政策分析规则
- 政策识别：识别政策类型、发文机关、效力层级
- 政策解读：提取核心内容、适用范围、执行标准
- 政策关联：分析政策关联性、配套政策
- 政策应用：评估政策适用性、申报条件

### 项目谋划规则
- 需求分析：明确项目类型、投资额、建设内容
- 政策匹配：匹配适用政策、资金渠道
- 方案设计：设计资金组合、申报策略
- 风险评估：识别风险、提供规避策略

### 文档格式规则
- 标题层级：使用Markdown标准格式
- 元数据：包含发文号、发文机关、发布日期
- 引用格式：使用标准引用格式
- 文件命名：遵循命名规范

## 十、系统进化机制

### 进化循环

```
用户输入素材/提问
      ↓
意图理解 ←── 使用已有零件
      ↓
工作流生成 ←── 参考已有流程
      ↓
成果产出 →── 输出成果
      ↓
零件提取 →── 自动提取新零件
      ↓
零件入库 →── 零件库扩充
      ↓
用户补齐 →── 用户补充完善
      ↓
系统优化 →── 系统能力提升
      ↓
返回顶部，循环进化
```

### 进化指标

| 指标 | 说明 |
|------|------|
| 总交互次数 | 用户输入素材/提问的总次数 |
| 成功率 | 成功产出成果的比例 |
| 零件增长 | 零件库中零件的增加数量 |
| 质量提升 | 输出成果质量的改进程度 |

## 十、Git工作流

### 分支策略
```
main (生产分支)
  ├── develop (开发分支)
  │   ├── feature/xxx (功能分支)
  │   └── hotfix (紧急修复分支)
```

### 提交规范
```
feat: 新增功能
fix: 修复bug
docs: 文档更新
refactor: 代码重构
test: 测试相关
chore: 构建/工具相关
```

### 自动化流程
```bash
# 完成任务后自动执行
git add .
git commit -m "feat: 完成{场景名称}"
git push origin main

# 创建版本标签
git tag -a v1.0.0 -m "版本1.0.0"
git push origin main --tags
```
