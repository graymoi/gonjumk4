# Trae 项目规则

## 文件夹命名规范

### 核心文件夹
```
项目根目录/
├── .trae/              # Trae配置
│   └── rules/          # 项目规则
├── core/               # 核心模块
├── adapters/           # 数据适配器
├── config/             # 配置文件
├── docs/               # 文档
├── hooks/              # 钩子系统
├── knowledge/          # 知识库
├── logs/               # 日志
├── outputs/            # 输出文件
├── parts/              # 可复用零件
├── rules/              # 业务规则
├── scripts/            # 脚本工具
├── tests/              # 测试文件
└── 00_政策法规/         # 政策库（数字前缀排序）
```

### 命名原则

1. **小写+下划线**: `core/`, `output_manager.py`
2. **数字前缀排序**: `00_政策法规/`, `01_按部门/`
3. **功能分类**: 按模块功能划分
4. **中英混合**: 业务文件夹可用中文

### 文件命名

1. **Python文件**: 小写+下划线 `intent_analyzer.py`
2. **配置文件**: 小写+下划线 `system_config.yaml`
3. **文档文件**: 日期+类型 `2026-03-12_项目谋划_报告.md`
4. **政策文件**: 发文号+机关+名称+日期

### 禁止事项

1. 禁止中文文件名（政策库除外）
2. 禁止空格和特殊字符
3. 禁止过长的文件名（>50字符）
4. 禁止无意义的命名

## 根目录原则

根目录只保留必要文件：
- `.gitignore` - Git配置
- `README.md` - 项目说明
- `main.py` - 主入口

其他文件按功能归入对应文件夹。
