# 文档格式规则

## 1. 文件命名规范

### 1.1 政策文件命名
**格式**：`[发文号]_[发文机关]_[政策名称]_[发布日期].md`

**示例**：
- `国办发〔2024〕52号_国务院办公厅_关于优化完善地方政府专项债券管理机制的意见_20241225.md`
- `建科〔2023〕45号_住房城乡建设部_关于推进建设工程质量检测机构资质许可工作的通知_20230510.md`

### 1.2 文档文件命名
**格式**：`[日期]_[类型]_[标题]_[版本]_[状态].md`

**示例**：
- `2026-02-21_政策解读_专项债券政策_V1.0_待审核.md`
- `2026-02-21_项目谋划_邢台站前广场_V1.0_已确认.md`

### 1.3 输出文件命名
**格式**：`[日期]_[场景]_[项目名称]_[文档类型].pdf`

**示例**：
- `2025-03-11_项目谋划_老旧小区改造_谋划报告.pdf`
- `2025-03-11_政策研究_专项债券_研究报告.pdf`

## 2. Markdown格式规范

### 2.1 标题层级
```markdown
# 一级标题（文档标题）
## 二级标题（章节）
### 三级标题（小节）
#### 四级标题（要点）
```

### 2.2 列表格式
```markdown
- 无序列表项1
- 无序列表项2
  - 子列表项1
  - 子列表项2

1. 有序列表项1
2. 有序列表项2
```

### 2.3 表格格式
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

### 2.4 引用格式
```markdown
> 引用内容
> 多行引用
```

### 2.5 代码格式
```markdown
`行内代码`

```python
# 代码块
print("Hello World")
```
```

## 3. 元数据规范

### 3.1 必填字段
| 字段 | 说明 | 示例 |
|------|------|------|
| file_number | 发文号 | 国办发〔2024〕52号 |
| publishing_agency | 发文机关 | 国务院办公厅 |
| policy_name | 政策名称 | 关于优化完善地方政府专项债券管理机制的意见 |
| publishing_date | 发布日期 | 20241225 |
| status | 状态 | 现行有效/已废止 |

### 3.2 可选字段
| 字段 | 说明 | 示例 |
|------|------|------|
| effective_date | 生效日期 | 20250101 |
| keywords | 关键词 | 专项债券, 地方政府 |
| business_domain | 业务领域 | 财政金融 |
| policy_type | 政策类型 | 规范性文件 |
| level | 层级 | 国家级/省级/市级 |

### 3.3 元数据模板
```yaml
---
file_number: 国办发〔2024〕52号
publishing_agency: 国务院办公厅
policy_name: 关于优化完善地方政府专项债券管理机制的意见
publishing_date: 20241225
effective_date: 20250101
status: 现行有效
keywords:
  - 专项债券
  - 地方政府
business_domain:
  - 财政金融
policy_type: 规范性文件
level: 国家级
---
```

## 5. 输出格式规范

### 5.1 Word文档
- 字体：正文宋体，标题黑体
- 字号：正文小四，标题三号
- 行距：1.5倍行距
- 页边距：上下2.54cm，左右3.17cm

### 5.2 PDF文档
- 保持Word格式
- 添加页码
- 添加目录
- 添加页眉页脚

### 5.3 PPT文档
- 标题字体：微软雅黑
- 正文字体：微软雅黑
- 配色：蓝色系
- 每页不超过6个要点

### 5.4 Excel文档
- 表头加粗
- 数据对齐
- 添加筛选
- 冻结首行

## 6. 引用规范

### 6.1 政策引用
```markdown
根据《政策名称》（发文号）规定，...
```

### 6.2 数据引用
```markdown
数据来源：[数据来源名称]，[发布时间]
```

### 6.3 文献引用
```markdown
[作者]. [文献名称]. [出版地]: [出版社], [年份].
```

## 7. 版本管理

### 7.1 版本号规则
- V1.0：初始版本
- V1.1：小修订
- V2.0：大修订

### 7.2 状态标识
- 草稿：正在编辑
- 待审核：等待审核
- 已确认：审核通过
- 已归档：不再修改

### 7.3 变更记录
```markdown
## 变更记录

| 版本 | 日期 | 修改人 | 修改内容 |
|------|------|--------|----------|
| V1.0 | 2025-03-11 | AI助手 | 初始版本 |
| V1.1 | 2025-03-12 | AI助手 | 修改XXX内容 |
```
