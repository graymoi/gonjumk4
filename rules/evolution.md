# 系统成长进化规则

## 1. 学习触发规则

### 1.1 自动学习触发条件
- 每次成功完成用户任务后
- 用户提供新的政策文件时
- 用户纠正AI输出时
- 用户添加新的项目案例时
- 用户反馈问题或建议时

### 1.2 学习内容提取
- 政策知识点
- 项目谋划经验
- 资金方案模板
- 风险案例
- 用户偏好

---

## 2. 零件库管理规则

### 2.1 零件类型
| 类型 | 说明 | 存储路径 |
|------|------|---------|
| 政策零件 | 政策要点、条款 | knowledge/parts/policies/ |
| 模板零件 | 文档模板、报告模板 | knowledge/parts/templates/ |
| 案例零件 | 项目案例、成功案例 | knowledge/parts/cases/ |
| 数据零件 | 统计数据、参数数据 | knowledge/parts/data/ |
| 流程零件 | 工作流程、操作步骤 | knowledge/parts/processes/ |
| 话术零件 | 回复话术、解释话术 | knowledge/parts/scripts/ |
| 风险零件 | 风险案例、风险提示 | knowledge/parts/risks/ |

### 2.2 零件提取规则
- **完整性**：零件应包含完整的上下文信息
- **独立性**：零件应可独立使用
- **可复用性**：零件应具有复用价值
- **标签化**：零件应有清晰的标签和分类

### 2.3 零件质量标准
- 准确性：信息准确无误
- 时效性：信息及时更新
- 相关性：与业务场景相关
- 实用性：具有实际应用价值

---

## 3. 进化机制规则

### 3.1 进化指标
```json
{
  "metrics": {
    "total_interactions": "总交互次数",
    "successful_outputs": "成功输出次数",
    "parts_added": "新增零件数",
    "quality_improvements": "质量改进次数",
    "user_satisfaction": "用户满意度",
    "knowledge_coverage": "知识覆盖率"
  }
}
```

### 3.2 进化等级
| 等级 | 条件 | 能力提升 |
|------|------|---------|
| Lv.1 | 初始状态 | 基础功能 |
| Lv.2 | 交互≥10次 | 解锁高级分析 |
| Lv.3 | 零件≥50个 | 解锁智能推荐 |
| Lv.4 | 成功率≥90% | 解锁自动谋划 |
| Lv.5 | 用户满意度≥95% | 解锁全自动化 |

### 3.3 进化触发
- 每日进化检查：每天凌晨自动检查进化条件
- 交互后进化：每次交互后检查是否满足进化条件
- 手动进化：用户可手动触发进化检查

---

## 4. 知识更新规则

### 4.1 政策更新
- 自动检测政策状态变化
- 及时更新政策内容
- 同步更新相关零件

### 4.2 案例更新
- 新增成功案例自动入库
- 失败案例提取教训
- 案例标签自动更新

### 4.3 模板更新
- 根据用户反馈优化模板
- 新增场景自动创建模板
- 模板版本管理
