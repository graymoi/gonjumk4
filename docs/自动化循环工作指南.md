# 自动化循环工作使用指南

> 城乡建设政策知识服务系统 - 自动化持续工作功能

---

## 概述

本系统实现了**自动化持续工作的Loop功能**，可以无需人工干预，自动执行循环任务，实现真正的"AI自主工作"。

---

## 核心组件

### 1. 自动循环引擎 (AutoLoopEngine)

**功能**：
- 持续循环执行预设任务
- 优先级任务队列管理
- 自动重试失败任务
- 状态持久化保存

**核心特性**：
```python
# 循环周期包含6个自动化任务
1. 政策监测 (policy_monitor)
2. 知识库更新 (knowledge_update)
3. 系统优化 (system_optimization)
4. 学习提取 (learning_extraction)
5. 健康检查 (health_check)
6. Git同步 (git_sync)
```

### 2. 工作流调度器 (WorkflowScheduler)

**功能**：
- 定时任务调度
- 任务依赖管理
- 资源优化分配
- 任务历史记录

**调度类型**：
- 定时任务：指定时间执行
- 周期任务：按间隔重复执行
- 依赖任务：等待前置任务完成

### 3. 持续学习引擎 (ContinuousLearningEngine)

**功能**：
- 从会话中提取学习模式
- 模式置信度动态调整
- 过期模式自动清理
- 高频模式生成技能

**学习模式类型**：
- 意图识别模式
- 工作流模板模式
- 技能使用模式
- 错误解决模式
- 优化改进模式
- 用户纠正模式

## 工作流编排

### 创建自定义工作流

```python
from core.workflow_scheduler import WorkflowOrchestrator

# 创建工作流
orchestrator.create_workflow(
    workflow_id="custom_workflow_1",
    name="自定义工作流",
    steps=[
        {
            "name": "步骤1: 政策监测",
            "type": "task",
            "task_type": "policy_monitor",
            "priority": 3
        },
        {
            "name": "步骤2: 并行处理",
            "type": "parallel",
            "tasks": [
                {"task_type": "knowledge_update", "priority": 5},
                {"task_type": "system_optimization", "priority": 4}
            ]
        },
        {
            "name": "步骤3: 条件判断",
            "type": "condition",
            "condition": {"type": "data_exists", "key": "result"},
            "then_step": {"name": "成功处理", "type": "task", "task_type": "git_sync"},
            "else_step": {"name": "失败处理", "type": "task", "task_type": "health_check"}
        }
    ]
)

# 执行工作流
instance_id = orchestrator.execute_workflow("custom_workflow_1", {"test": True})
```

## 配置说明

### 循环间隔配置

```python
# 在 main.py 中修改
self.auto_loop_engine.start(interval=120)  # 120秒间隔
```

### 学习规则配置

```python
# 在 core/continuous_learning.py 中修改
self.learning_rules = {
    "min_confidence": 0.3,        # 最低置信度
    "confidence_boost": 0.1,      # 置信度提升
    "confidence_decay": 0.05,     # 置信度衰减
    "min_usage_for_skill": 5,     # 生成技能的最小使用次数
    "success_threshold": 0.7,     # 成功率阈值
    "pattern_expiry_days": 90     # 模式过期天数
}
```

### 定时任务配置

```python
# 在 core/auto_loop_engine.py 中修改
def _check_scheduled_tasks(self):
    now = datetime.now()
    
    # 每天9点生成日报
    if now.hour == 9 and now.minute < 5:
        self.add_task("daily_report", "report_generation", priority=3)
    
    # 每天18点Git同步
    if now.hour == 18 and now.minute < 5:
        self.add_task("daily_summary", "git_sync", priority=4)
    
    # 每周一8点系统优化
    if now.weekday() == 0 and now.hour == 8:
        self.add_task("weekly_optimization", "system_optimization", priority=2)
```

## 最佳实践

### 1. 合理设置循环间隔

- **高频监测**: 30-60秒（政策动态、紧急任务）
- **常规监测**: 60-300秒（知识库更新、系统优化）
- **低频监测**: 300-600秒（健康检查、备份）

### 2. 任务优先级设置

- **优先级 1-2**: 紧急任务（系统故障、错误修复）
- **优先级 3-4**: 重要任务（政策监测、日报生成）
- **优先级 5-6**: 常规任务（知识更新、优化）
- **优先级 7-10**: 低优先级（清理、备份）

### 3. 学习模式管理

- 定期查看学习报告
- 手动进化模式（每周一次）
- 检查生成的技能
- 清理低质量模式

### 4. 监控与告警

- 定期检查系统健康状态
- 关注任务失败率
- 监控资源使用情况
- 设置关键任务告警

## 进阶用法

### 自定义自动化任务

```python
# 添加自定义任务类型
def custom_task_handler(data: Dict) -> Dict:
    # 自定义逻辑
    return {"status": "success"}

# 注册到循环引擎
auto_loop_engine.auto_tasks["custom_task"] = custom_task_handler
```

### 集成外部系统

```python
# 集成Webhook通知
def notify_webhook(result: Dict):
    import requests
    requests.post("https://your-webhook-url", json=result)

# 添加回调
auto_loop_engine.add_task(
    task_id="notify_task",
    task_type="policy_monitor",
    callback=notify_webhook
)
```
