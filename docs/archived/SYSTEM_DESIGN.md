# 城乡建设政策知识服务系统 - 系统设计文档

> **设计理念**：可生长、自动进化的AI自动化系统
> **核心目标**：将政策洞察转化为可执行的项目方案，解锁下游设计和施工合同

---

## 一、系统定位

### 我们是谁
城乡建设咨询领域的政策知识服务系统，由用户与AI智能体协作运行。

### 核心目标
**最终目标**：将政策洞察转化为可执行的项目方案，解锁下游设计和施工合同

**价值链**：
```
政策采集 → 政策解读 → 项目谋划 → 资金匹配 → 方案输出 → 设计合同
```

---

## 二、设计原则

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| **场景无关** | 不依赖预设场景，AI理解用户意图 | 意图识别 + 动态场景生成 |
| **工作流动态** | 根据需求AI生成工作流 | 工作流编排引擎 |
| **知识自学习** | 从每次交互中学习优化 | 反馈学习机制 |
| **技能可扩展** | 随时添加新skills | 技能插件系统 |
| **数据源开放** | 可接入任意数据源 | 数据适配器模式 |
| **全自动化** | AI一次性完成，用户仅审核 | 智能工作流 |

## 四、核心模块设计

### 4.1 意图理解与场景生成器

**功能**：
- 理解用户输入的真实意图
- 动态生成场景描述
- 识别所需技能和数据

**实现方式**：
- 利用现有skills：`brainstorming`、`scientific-critical-thinking`
- 使用LLM进行意图分析
- 场景模板库（可扩展）

**输入输出示例**：
```json
输入："我想谋划一个老旧小区改造项目，大概投资5000万"

输出：
{
  "intent": "项目谋划",
  "scenario": {
    "name": "老旧小区改造项目谋划",
    "type": "项目谋划",
    "requirements": {
      "投资额": "5000万",
      "项目类型": "老旧小区改造",
```

```python
def update_readme():

    # 1. 收集系统信息

    workflows = get_workflow_stats()
```

### 10.5 零件自动提取

**生产过程中自动提取零件**：

```python
# 示例：从项目谋划过程中提取零件

def extract_parts_from_workflow(workflow_result):
    """从工作流结果中提取零件"""
    
    parts = []
    
    # 1. 提取政策零件
    for policy in workflow_result.get("matched_policies", []):
        part = {
            "type": "policy",
            "name": policy["name"],
            "content": policy["key_points"],
            "source": "auto_extracted",
            "related_scenario": workflow_result["scenario"]
        }
        parts.append(part)
    
    # 2. 提取模板零件
    if workflow_result.get("report_template"):
        part = {
            "type": "template",
            "name": f"{workflow_result['scenario']}模板",
            "content": workflow_result["report_template"],
            "source": "auto_generated"
        }
        parts.append(part)
    
    # 3. 提取流程零件
    if workflow_result.get("workflow_steps"):
        part = {
            "type": "process",
            "name": f"{workflow_result['scenario']}流程",
            "content": workflow_result["workflow_steps"],
            "source": "auto_extracted"
        }
        parts.append(part)
    
    return parts
```
