# 00_政策法规

## 文件夹用途
本文件夹用于存放与城乡建设领域相关的**法律法规、政府规章、规范性文件、指导意见、部门文件**等，是政策咨询、项目谋划和资金申报的核心依据来源。所有文件应为官方发布的正式版本，确保权威性和准确性。

## 内部资料管理

### 存储位置
内部资料统一放在 `00_政策法规/国家级/内部资料/` 文件夹中，与公开文件物理分开。

### 访问控制
- 在文件系统层面设置严格的访问控制
- 建议加密存储
- 仅限内网访问
- 记录访问日志

### 文件类型
1. **未发布文件**：征求意见稿、草案、待发布文件
2. **内部文件**：会议纪要、内部通知、工作文件、研究报告

### 命名规范
- **未发布文件**：前缀 `[未发布]`
  - 示例：`[未发布]2025年城市更新专项资金申报指南（征求意见稿）.md`
- **内部文件**：前缀 `[内部]`
  - 示例：`[内部]关于XX片区规划的会议纪要.md`

### 元数据规范
内部文件必须包含 `classification: 内部` 字段：

```yaml
---
title: 文件标题
file_number: 文号（如有）
publishing_agency: 发布部门
publishing_date: 发布日期
document_type: 文件类型
classification: 内部  # 必须标记为内部
effectiveness: 有效/未发布/征求意见稿
---
```

### AI检索说明
当AI进行检索时，会根据用户权限过滤结果：
- 即使文件在内部文件夹，也需确保用户有权访问
- 内部文件不会向未授权用户展示

## 政策解读

### 存储位置
政策解读文件存放在 `00_政策法规/政策解读/` 目录下，按主题分类管理。

### 文件特点
政策解读文档具有以下特点：
1. **基于政策原文**：所有解读都基于正式发布的政策文件，确保准确性
2. **非原文性质**：这些文档不是政策原文，而是对政策的解读和分析
3. **需要原文支撑**：解读内容需要引用政策原文作为依据
4. **实用导向**：解读注重政策应用和实际操作

### 与政策原文的关系
- **政策原文**：存放在 `00_政策法规/国家级/按部门/` 和 `00_政策法规/国家级/按领域/` 目录下
- **政策解读**：存放在 `00_政策法规/政策解读/` 目录下，是对政策原文的补充说明

### 使用建议
1. 先阅读政策原文，了解政策的基本内容和要求
2. 再查阅政策解读，深入理解政策的具体应用和操作要点
3. 结合实际工作，将政策解读内容应用到具体项目中

### 注意事项
- 政策解读仅供参考，具体政策执行以官方发布的政策原文为准
- 如有疑问，应以政策原文或官方解释为准
- 政策解读会根据政策更新及时修订

## 使用场景

### 政策咨询
用户问"城市更新有哪些国家级政策？"，AI从国家级/规范性文件/城市更新下检索相关文件，并返回摘要和原文。

### 资金匹配
系统根据项目类型，从政策库中筛选出可能支持该项目的政策，并提取其中的资金支持条款。

### 项目谋划
策划人员查看某领域政策趋势，了解国家鼓励方向，生成项目创意。

---

## 维护与更新

- **定期检查**：每季度对政策状态进行复核，将已废止文件移入"已废止"目录
- **版本管理**：若同一政策有修订版本，建议保留旧版，并注明"已被XX号文替代"
- **人工复核**：AI自动处理的结果需由知识库管理员定期抽查

- 相同部门发布的政策通过标签聚集
- 使用 `#发改委`、`#财政部`、`#住建部` 等标签筛选

#### 2. 按类型联系
- 相同类型的政策通过标签聚集

#### 3. 按时间联系
- 使用发布时间进行排序

#### 4. 按关联联系
- 相关政策之间建立链接关系
- 使用 `related_policies` 字段建立政策关联
