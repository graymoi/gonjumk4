## 🔗 链接关系建立

### 相关政策链接

使用 `related_policies` 字段建立政策之间的关联关系：

```yaml
related_policies:
  - 发改环资规〔2024〕338号
  - 政府投资条例
  - 中央预算内投资项目监督管理办法
```

**说明**：
- 列出与当前政策相关的其他政策
- 可以使用发文号或政策名称
- 便于在图谱视图中查看关联关系

### 知识链接

使用 `knowledge_links` 字段建立与知识库其他部分的链接：

```yaml
knowledge_links:
  - [[政策知识链路和关联关系]]
  - [[知识图谱_资金政策]]
  - [[政策文件汇总_知识链接]]
```

**说明**：
- 使用 Obsidian 的双向链接语法 `[[文件名]]`
- 链接到知识库中的其他文档
- 便于在图谱视图中查看知识关联

## 📝 添加元数据的步骤

### 步骤1：打开政策文件

在 Obsidian 中打开需要添加元数据的政策文件。

### 步骤2：添加元数据

在文件开头添加 YAML front matter 元数据块：

```yaml
---
title: 政策标题
file_number: 发文字号
publishing_agency: 发布机构
publishing_date: 发布日期
...
---
```

### 步骤3：填写字段

根据元数据模板填写字段：

1. **必填字段**：title, file_number, publishing_agency, publishing_date, document_type, level, effectiveness, category, tags
2. **可选字段**：supervising_agency, online_url, related_policies, knowledge_links, effective_date, expiry_date, funding_type, project_type, region, amount_limit

### 步骤4：建立链接

建立政策之间的链接关系：

1. **相关政策**：在 `related_policies` 字段中添加相关政策
2. **知识链接**：在 `knowledge_links` 字段中添加知识库链接

### 步骤5：更新索引

更新相应的索引文件：

1. **国家级政策索引**：添加到相应部门分类
2. **类型索引**：添加到相应类型分类
3. **时间索引**：添加到相应时间分类

## 🎯 最佳实践

### 1. 保持一致性

- 使用统一的命名规范
- 使用统一的标签体系
- 使用统一的元数据格式

### 2. 建立关联

- 建立政策之间的关联关系
- 建立政策与知识库的链接
- 定期检查链接的有效性

### 3. 定期更新

- 定期更新元数据信息
- 定期更新政策索引
- 定期清理废止政策

### 4. 使用标准标签

- 使用标准标签体系
- 避免自定义标签
- 定期清理无用标签
